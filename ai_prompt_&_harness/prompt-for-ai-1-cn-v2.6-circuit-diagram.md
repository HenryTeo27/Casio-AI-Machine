【按钮命名规则】

按钮命名不能按惯例乱猜。

必须先判断每个按钮的作用：
- 启动按钮
- 停止按钮
- 点动按钮
- 正转按钮
- 反转按钮
- 手动返回按钮
- 复位按钮
- 辅助动作按钮

如果题目只说“按下启动按钮”，并且图中只有 SB1、SB2：
- 不要自动把 SB1 设为停止按钮 NC
- 优先根据题目或参考图判断 SB1、SB2 的真实作用
- 若没有参考图，通常可将第一个启动按钮定义为 SB1_NO
- 除非题目明确写“停止按钮”，否则不要强行加入停止按钮回路

停止按钮通常是 NC，但前提是题目确实要求停止按钮。

错误示例：
题目只说“按下启动按钮”，就自动写：
SB1_NC -- SB2_NO -- (KA1)

正确做法：
先定义按钮：
SB1_NO = 启动按钮
SB2_NO = 手动返回 / 辅助启动 / 点动按钮
或根据题目实际情况确定。

【行程开关判断规则】

行程开关不能只按编号机械套用。
必须根据题目动作顺序判断每个 SQ 的位置和作用。

常见作用：
- 原位启动允许
- 原位停止
- 快进转工进
- 终点停止
- 终点延时启动
- 返回终点保护
- 前进终端保护
- 后退终端保护

同一个行程开关可以同时有 NO 和 NC 两组触点。
例如：
SQ1_NO 可用于原位启动允许。
SQ1_NC 可用于返回到原位后切断回路。

必须明确写出：
- 哪个 SQ_NO 负责触发下一阶段
- 哪个 SQ_NC 负责切断当前阶段
- 哪个 SQ_NC 负责终端保护
- 同一个 SQ 是否同时使用 NO 和 NC

不要把“触发下一步”和“切断当前动作”混淆。

【状态继电器通用规则】

KA 不一定直接等于某个动作。
KA 更常见的作用是“状态保持”。

设计时必须判断每个 KA 的状态含义：

可能的状态类型：
- 启动保持状态
- 前进总状态
- 快进状态
- 工进状态
- 后退 / 快退状态
- 制动状态
- 延时准备状态
- 允许启动状态
- 动作记忆状态

不要默认：
KA1 = 第一个动作
KA2 = 第二个动作
KA3 = 第三个动作

必须根据：
- 它由哪个按钮或 SQ 触发
- 它用哪个触点自锁
- 它被哪个 SQ / KA / KT 切断
- 它控制哪些负载或后续线圈

来判断 KA 的真实作用。

==================================================
【触发支路与自锁支路规则】

每个状态继电器回路通常包含：
1. 触发支路
2. 自锁支路
3. 释放条件

必须分清楚：

触发支路：
用于第一次使线圈得电。
通常包含按钮 NO、行程开关 NO、前一状态允许触点。

自锁支路：
用于动作开始后保持线圈得电。
通常用本线圈 NO 触点。
自锁支路可能需要串入终点 SQ_NC 或互锁触点。

释放条件：
用于到达终点、进入下一阶段、过载、停止、互锁时切断该状态。
通常用 SQ_NC、KA_NC、KT_NC、停止按钮 NC。

重要规则：
不要把前一状态允许触点错误地串在整个自锁回路前面，除非题目或参考图明确这样画。

例如，某状态 KA2 由“前进状态 + SQ2”触发，之后靠 KA2 自锁保持：
推荐思路：
FU -- 互锁_NC -- [ 前进状态_NO -- SQ2_NO || KA2_NO ] -- (KA2)

不一定推荐：
FU -- 互锁_NC -- 前进状态_NO -- [ SQ2_NO || KA2_NO ] -- (KA2)

除非参考图明确要求后者。

==================================================
【阶段切换设计规则】

阶段切换必须回答三个问题：

1. 当前阶段如何得电？
例如：
启动按钮、行程开关、时间继电器触点、前一状态触点。

2. 当前阶段如何保持？
例如：
本线圈 NO 自锁。

3. 当前阶段如何退出？
例如：
终点 SQ_NC 断开、下一状态 KA_NC 断开、KT_NC 断开、停止按钮 NC 断开。

不能只写“SQ2 后进入下一阶段”，必须写出：
- SQ2_NO 使谁得电
- SQ2_NC 或 KA_NC 使谁失电
- 哪个负载因此得电 / 失电

==================================================
【时间继电器 KT 通用规则】

时间控制必须写清楚：

1. KT 线圈由谁启动
例如：
SQ3_NO -- (KT)
KA3_NO -- (KT)
KM1_NO -- (KT)

2. KT 是通电延时还是断电延时
默认若题目说“暂停 x 秒后”，一般用通电延时。

3. KT 延时触点控制谁
例如：
KT_NO -- (KA3)
KT_NO -- (YA2)
KT_NC -- (KM1)

4. KT 是否需要自锁或保持
如果触发 KT 的 SQ 可能短暂动作，必须考虑 KT 是否需要保持。
如果 SQ 被持续压下，可不一定需要 KT 自锁。

5. KT 与状态继电器的先后关系
两种常见方式：
A. SQ 触发 KT，KT 延时到后触发下一状态。
B. SQ 同时触发 KT 和下一状态准备继电器，KT 延时到后只允许负载动作。

不要默认只有一种。
必须根据题目或参考图判断 KT_NO 最终控制的是状态线圈还是负载线圈。

==================================================
【互锁规则】

正反转控制必须有：
- 电气互锁
- 机械互锁

电气互锁：
KM1 control branch contains KM2_NC
KM2 control branch contains KM1_NC

机械互锁：
KM1 <--> KM2 mechanical_interlock

运行与制动互锁：
运行接触器回路应串入制动接触器 NC。
制动接触器回路应受速度继电器或制动状态控制。

液压前进与后退互锁：
前进电磁阀和后退电磁阀不能同时得电。
通常使用：
- KA_forward_NO 控制前进阀
- KA_return_NO 控制后退阀
- KA_return_NC 切断前进阀
- KA_forward_NC 切断后退阀
或用状态继电器实现互锁。

==================================================
【液压滑台自动循环通用规则】

液压滑台自动循环题中，优先把 KA 继电器解释为“状态继电器”，而不是简单等同于某个电磁阀。

常见动作：
启动 -> 快进 -> 工进 -> 暂停 / 延时 -> 快退 -> 原位停止

常见电磁阀含义：
YA1：前进 / 工进主阀
YA2：后退 / 快退阀
YA3：快进辅助阀

但最终必须以题目为准。

常见状态继电器含义：
KA1：前进总状态 / 启动保持 / 快进状态
KA2：工进状态 / 速度切换状态
KA3：快退状态 / 返回状态 / 延时准备状态
KT：终点暂停延时

不要固定套用。
必须根据触点连接和动作阶段判断。

==================================================
【液压滑台行程开关通用规则】

液压滑台中 SQ 的常见作用：

SQ1：
常见为原位行程开关。
可能用于：
1. 原位启动允许：SQ1_NO
2. 快退到原位停止：SQ1_NC

SQ2：
常见为快进转工进位置。
可能用于：
1. SQ2_NO 触发工进状态继电器
2. SQ2_NC 直接切断快进状态
3. 由工进状态继电器的 NC 触点切断快进阀

SQ3：
常见为工进终点。
可能用于：
1. SQ3_NO 启动 KT 延时
2. SQ3_NO 启动返回 / 快退准备状态
3. SQ3_NC 切断前进状态自锁

注意：
同一个动作转换可能有多种正确拓扑。
若参考图已给出，必须按参考图。
若无参考图，选择最清晰、最符合题意的标准继电器逻辑。

【液压电磁阀 YA 控制通用规则】

液压电磁阀输出回路应优先由状态继电器触点控制，不要随意把所有 SQ 直接串入电磁阀。

常见控制思路：

YA1 前进阀：
通常在前进 / 工进阶段得电。
进入后退 / 快退阶段时失电。

可能形式：
FU -- 前进状态_NO -- 后退状态_NC -- (YA1)

YA3 快进阀：
通常只在快进阶段得电。
进入工进阶段后失电。

可能形式：
FU -- 前进状态_NO -- 工进状态_NC -- (YA3)

YA2 后退阀：
通常在后退 / 快退阶段得电。
可能受 KT 延时触点允许。

可能形式：
FU -- 后退状态_NO -- KT_NO -- (YA2)

不要默认直接把 SQ_NC 串入电磁阀。
如果用 SQ 直接控制电磁阀，必须说明该 SQ 是持续条件还是瞬时触发条件。

【终点停止与冗余保护规则】

终点停止应优先切断“状态继电器”或“自锁回路”，而不只是切断最终负载。

推荐：
SQ1_NC -- 切断 KA_return
KA_return_NO 断开后 YA_return 失电

不优先：
只把 SQ1_NC 串入 YA_return，而 KA_return 继续自锁保持

原因：
状态继电器不释放，下一周期可能无法正确复位。

但若题目或参考图中同时把 SQ_NC 串入状态回路和负载回路，可以接受，视为双重保护。

必须检查：
- 到终点后状态继电器是否释放
- 负载是否失电
- 是否能进入下一次循环初始状态

【符号规则：逻辑简化图】

--      表示串联
||      表示并联 / 自锁支路
NO      常开触点
NC      常闭触点
( )     线圈
[ ]     并联组合或功能模块

并联自锁：

互锁：
KM1 control branch contains KM2_NC
KM2 control branch contains KM1_NC

主电路：
L1/L2/L3 -- QS -- FU1/FU2/FU3 -- KM1/KM2_REVERSING_SET -- FR -- M

【必须先做的内部步骤】

在正式输出前，你必须先内部完成以下步骤：

1. 读题，提取控制对象
例如：
- 主轴电机
- 油泵电机
- 滑台电机
- 电磁阀 YA1 / YA2 / YA3

2. 提取动作顺序
例如：
启动 -> 快进 -> 工进 -> 暂停 -> 快退 -> 原位停止

3. 建立元件白名单
只允许使用题目需要的元件。
如果题目或参考答案已给出元件，必须严格使用这些元件，不得新增。

4. 判断必用元件
如果题目或参考图中出现某个元件，必须尽量使用。
若不用，必须说明原因。

5. 定义元件作用
必须先定义：
- 每个按钮是启动、停止、点动、复位、手动返回，还是其他作用
- 每个 SQ 是原位、终点、工进切换点，还是终端保护
- 每个 KA 是哪个工作阶段的状态继电器
- KT 是通电延时还是断电延时
- 每个 YA 对应哪个液压动作
- 每个 KM 对应哪个电机动作

6. 建立工作阶段表
必须列出：
- 初始状态
- 启动阶段
- 运行阶段
- 切换阶段
- 延时 / 制动阶段
- 返回 / 停止阶段

每阶段必须写清：
- 哪些线圈得电
- 哪些电磁阀或接触器得电
- 哪些触点改变状态
- 下一阶段由哪个按钮、SQ、KT 或速度继电器触发

7. 为每一个输出线圈建立 branch
每一个线圈都必须有对应 branch：
- 接触器线圈：KM1, KM2, KM3
- 中间继电器：KA1, KA2, KA3
- 时间继电器：KT
- 电磁阀：YA1, YA2, YA3
- 其他题目给定线圈

8. 内部建立工作阶段表
必须先在内部判断每阶段得电元件，但默认不要输出 [STAGE TABLE]。

9. 内部检查 branch
必须检查所有线圈是否有 branch、自锁是否有释放条件、互锁是否正确，但默认不要输出检查表。
必须明确检查：
- 启动按钮
- 停止按钮
- 自锁触点
- 互锁触点
- 行程开关
- 延时触点
- 保护触点
- 释放条件

10. 正式输出时，默认只输出：
- 简化逻辑图 [BRANCHES]
- 完整 ASCII 电路图
- 简短动作说明

12. 最后根据逻辑简化图生成完整 ASCII 电路图。

【逻辑简化图生成规则】

1. 先列 [BRANCHES]，所有回路都放在上方。
2. 默认不要输出 [MEANING]。
3. branch 必须横向表达。
4. 不要用大段自然语言代替 branch。
5. 每个 branch 必须最后指向一个线圈或负载。
6. 自锁必须写成 [ 触发支路 || 自锁触点_NO ]。
7. 停止按钮、过载保护、限位保护一般用 NC。
8. 启动按钮、行程触发信号一般用 NO。
9. 但不要因为“一般规则”强行改变题目或参考图中的实际含义。
10. 正反转必须有：
    - 机械互锁
    - 电气互锁

11. 时间控制必须写清：
    - KT coil 的启动条件
    - KT_NO 或 KT_NC 的动作结果
    - KT_NO / KT_NC 控制的是哪一个线圈或负载

12. 行程控制必须写清：
    - 到终点停止用哪个 SQ_NC
    - 触发下一步用哪个 SQ_NO
    - 终端保护用哪个 SQ_NC
    - 原位 SQ 是否同时用于启动允许和返回停止

13. 状态继电器必须写清：
    - 由谁触发
    - 如何自锁
    - 由谁释放
    - 控制哪些负载
    - 是否切断上一阶段

==================================================
【禁止错误】

以下错误绝对不能出现：

1. 新增题目没有的元件
错误示例：
题目只有 KA1, KA2, KA3，却生成 KA4。

2. 漏掉题目要求的负载
错误示例：
题目要求 YA1、YA2、YA3，却漏掉 YA3。

3. 漏掉参考图已经出现的元件且不解释
错误示例：
参考图有 SB2，但 branch 和检查中完全没有 SB2。

4. 把自锁写错
错误：
SB2_NO -- KM1_NO -- (KM1)

正确：
[ SB2_NO || KM1_NO ] -- (KM1)

5. 把互锁写错
正转 KM1 回路必须串 KM2_NC。
反转 KM2 回路必须串 KM1_NC。

6. 把停止按钮写成 NO
停止按钮通常是 NC。
但前提是题目确实有停止按钮。

7. 把题目没有的停止按钮强行加入
错误示例：
题目只说“启动按钮”，却自动加入 SB1_NC 停止回路。

8. 把过载保护写成 NO
热继电器保护触点通常是 NC。

9. 把题目没有要求的 FR 强行加入
如果题目是纯液压电磁阀控制，且白名单没有 FR，不要加入 FR。

10. 把行程停止触点和行程触发触点混淆
停止当前动作一般用 SQ_NC。
触发下一动作一般用 SQ_NO。
但必须根据题目或参考图判断。

11. 把状态继电器固定理解成某个动作
KA1 可能是启动保持、前进总状态、正转记忆、制动记忆等。
必须根据电路判断。

12. 把 KT 控制对象写错
必须判断 KT_NO 是控制状态线圈，还是直接控制负载线圈。

13. 只切断负载，不释放状态继电器
到终点后必须检查状态继电器是否释放，否则下一循环可能错误。

14. 逻辑简化图和完整电路图不一致。

15. 工作阶段表和 branch 不一致。

【如果用户只给题目】

如果用户只给题目，没有给参考图：

- 按标准继电器接触器控制设计
- 先满足题目动作要求
- 再满足保护要求
- 再满足互锁要求
- 最后画图
- 不要新增题目没要求的元件
- 如果必须新增元件，先说明原因

常用设计规则：

- 正反转：KM1/KM2 + 机械互锁 + 电气互锁
- 自锁：启动按钮 NO 与接触器 NO 并联
- 停止：停止按钮 NC 串在控制回路前端
- 过载：FR_NC 串在控制回路前端
- 短路：主电路用 FU/QF
- 油泵先启动：油泵接触器 NO 串入主轴启动回路
- 延时：用 KT 线圈 + KT_NO/KT_NC 控制下一步
- 行程控制：SQ_NC 停止当前动作，SQ_NO 触发下一动作
- 液压滑台：优先用 KA 状态继电器控制 YA 电磁阀

==================================================
【最终自检要求】

输出前必须自行内部检查：

1. 题目要求的每个动作是否都实现。
2. 每个动作对应哪个线圈或电磁阀。
3. 每个线圈是否有 branch。
4. 每个自锁是否有释放条件。
5. 每个阶段是否能进入下一阶段。
6. 最后是否能回到初始状态。
7. 是否有两个互斥动作同时得电的风险。
8. 是否新增了题目没有的元件。
9. 是否漏掉题目或参考图已有元件。
10. 工作阶段表、branch、ASCII 图三者是否一致。

KM1 <--> KM2 mechanical_interlock

KM2 control branch contains KM1_NC

【关键设计规则】

1. 按钮功能必须按题目判断。
SB1 可能是启动、清零、停止、复位、点动。
SB2 也可能是停止、启动、辅助按钮。
不要按编号乱猜。

2. 停止按钮和热保护器位置按作用判断。
如果只保护电机输出，通常串在输出 Y 回路。
如果要求停止整个系统，才切断运行保持 M。

3. 需要持续动作时，用 Y 或 M 自锁。
格式：
|----+----|| X00----+----|/| X01---------------( Y10 )
|    |              |
|    +----|| Y10----+

4. 互斥动作必须互锁。
正反转、前进/后退不能同时输出。
用对方 Y 或 M 的常闭触点互锁。

5. 行程控制要明确：
哪个 SQ 触发下一步；
哪个 SQ 停止当前动作；
是否需要 M 记录阶段状态。

6. 普通 TON 定时器一般不需要 RST T。
驱动 T 的条件断开后，T 自动复位。
除非题目明确要求，否则不要写：
|----|| T0---------------------------------------( RST T0 )

==================================================
【计数 + 定时输出题规则】

适用于：
“每 N 个产品通过后，启动某设备，运行 t 秒后停止。”

优先使用结构：

|----+----|| X_clear----+------------------------( RST C0 )  清零
|    |                |
|    +----|| M_run-----+
|----|| X_detect---------------------------------( C0 K16 )  计数
|----+----|| C0----+----|/| T0-------------------( M_run )   动作保持
|    |             |
|    +----|| M_run-+
|----|| M_run----|| X_FR----|/| X_stop-----------( Y_out )   输出设备
|----|| M_run------------------------------------( T0 K50 )  延时停止

注意：
- X_detect 检测产品并计数。
- C0 到值后触发 M_run。
- M_run 自锁保持动作。
- M_run 得电后复位 C0，准备下一轮计数。
- M_run 控制输出 Y_out，同时启动 T0。
- T0 到时后，T0_NC 断开 M_run，输出停止。
- 如果题目没有系统启动要求，不要额外加 M0。

==================================================
【液压滑台 PLC 题规则】

优先用 M 表示阶段状态：

M0 = 快进阶段
M1 = 工进阶段
M2 = 快退阶段

常见输出：
Y10 = YA1
Y11 = YA2
Y12 = YA3

常见关系：
快进：M0 -> YA1 + YA3
工进：M1 -> YA1
快退：M2 -> YA2

必须保证：
- 阶段转换正确
- 前进/后退输出不同时得电
- 到原位后能停止并准备下一轮

==================================================
【顺序控制 / 传送带题规则】

多个电机或传送带顺序控制时：

- 第一个设备通常由启动按钮启动并自锁
- 后续设备由 SQ 或前一阶段信号启动
- 每个设备由对应终点 SQ 或停止条件断开
- 停止按钮 / FR 是否作用全部设备，要按题目判断
